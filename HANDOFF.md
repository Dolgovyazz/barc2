## О проекте
Лендинг **BARCODE** — онлайн-библиотека барного дела в Telegram (рецепты коктейлей, виды алкоголя, техники). Язык — русский. Тёмная сине-навигационная «барная» эстетика (чёрный фон, синий `#1a6fff`, голубой `#a8d4ff`, aurora-свечения, глянцевые 3D-звёзды).

- **Путь (корень проекта):** `C:\Users\SystemX\Desktop\barcode\web` — здесь `app/`, `package.json`, отсюда запускать `pnpm dev`. HANDOFF лежит тут же (`web/HANDOFF.md`).
- **Стек:** Next.js 16.2.6 (App Router, **Turbopack**), React 19, Tailwind CSS 4, TypeScript 5.7, pnpm.
- **Запуск:** `pnpm dev` (порт 3000). Проверка: `npx tsc --noEmit`, `npx next build`.
- Много инлайн-стилей (наследие из v0.app) + CSS-переменные в `globals.css`.

## Текущее состояние (актуальный вид — читать первым)
Лендинг допилен по многим итерациям фидбека (последний большой — Раунд 6: настоящее стекло, декор, адаптив под ultrawide, центровки). Как сейчас выглядят секции:

Три глянцевые звезды (`GlossyStar`) обрамляют контент на ФИКС-отступе `calc(50% - 512px)` от центра (88px top-left + две по 66px снизу). Контурный бар-декор (бокалы/бутылка/«+/×») раскидан по всей ширине hero — вьюпорт-пропорционально (см. нюанс про адаптив декора). Кнопка «Перейти к обучению» скроллит к тарифам.
- **About (#about):** левая текст-карта — МАТОВОЕ СТЕКЛО (`rgba(10,22,46,.62)` + радиальный синий блик снизу-слева + инлайн `blur(12px)`). За карточками слоем НИЖЕ (`zIndex:0`) — ТРИ декоративных самолёта `plane.webp` (классы `.about-plane--left/center/right`, ~185/130/185px), НОСАМИ ВНУТРЬ к контенту (лев→вправо `rotate(-10)`, центр→вниз `rotate(80)`, прав→влево `rotate(170)`). Старая декор-звезда `star-v2.webp` и одиночный самолёт УДАЛЕНЫ. Справа сетка 2×2 СТЕКЛЯННЫХ feature-тегов (`blur(12px)`) — сквозь них ПРОСВЕЧИВАЮТ самолёты мягким синим фростом + кнопка «Связаться со мной».
- **For-whom (#for-whom):** «BARCODE подойдёт» — 5 карточек одинаковой ширины (3 сверху + 2 снизу по центру).
- **Pricing (#pricing):** карточка «Бармен» — МАТОВОЕ СТЕКЛО (`blur(14px)`) + СВЕЧЕНИЕ из верх-правого угла (радиал в `background`) + внешний ореол (`box-shadow`). Список фич: «Тесты и шпаргалки» ВТОРЫМ, «Авторские коктейли» ПОСЛЕДНИМ с золотой плашкой **ЭКСКЛЮЗИВ** (инлайн-span). Флангируют молнии `bolt.webp`, за заголовком кольцо `ring.webp`. Вокруг карточки — контурный бар-декор в полосе `min(1340px)`. Модалка «Подробнее» пока НЕ стеклянная (`var(--bg-card)`).
- **Nav:** ссылки АБСОЛЮТНО отцентрированы по странице (`md:absolute md:left-1/2 md:-translate-x-1/2`) — иначе `justify-between` при разной ширине лого (41px) / соцсетей (72px) уводил их влево. Иконки Instagram/Telegram 30px.
- **Всё «стекло» — через ИНЛАЙН `backdropFilter`** (Lightning CSS вырезает его из globals.css). Блюры сейчас УМЕРЕННЫЕ (12–16px), чтобы за стеклом читались фигуры/самолёты (референс заказчика). Раньше блюр не рендерился ВООБЩЕ — см. нюанс №3 про will-change.

**Проверки:** `tsc --noEmit` чистый; консоль без ошибок; проверено на 1280 и 2560 (ultrawide) через Claude-in-Chrome; горизонтального оверфлоу нет.

### Раунд 6 — настоящее стекло, декор, адаптив под ultrawide, центровки
- **КРИТИЧНО — блюр стекла НЕ РЕНДЕРИЛСЯ.** `.reveal` (scroll-анимация) держал `will-change: opacity, transform` НАВСЕГДА → в Chromium предок с `will-change`/`transform` делает `backdrop-filter` у потомков no-op (пустой backdrop). Поэтому подъём блюра 14→22→44px не давал НИЧЕГО (заказчик: «как будто не поменялось»). Фикс: `.reveal-ready .reveal.reveal-in { will-change: auto }`. `getComputedStyle` при этом ЛЖЁТ — показывает `blur(44px)`, а на экране ноль. Диагностика: обойти предков на `transform/filter/perspective/will-change ≠ none/auto`.
- **Стекло докручено под референс:** блюры снижены до умеренных (about 12, feature-теги 12, тариф 14, nav 16) + тон стекла сделан полупрозрачным (about `rgba(10,22,46,.62)`+блик), чтобы за стеклом ПРОСВЕЧИВАЛИ фигуры.
- **Самолёты в about переделаны:** убраны декор-звезда + одиночный самолёт; добавлены 3 самолёта (`.about-plane--left/center/right`) слоем за карточками, носами внутрь, крупнее (150/130/150).
- **Свечение на «Бармен»:** радиал из верх-правого угла в `background` карточки + внешний ореол в `box-shadow`.
- **Декор hero и pricing:** `Plus` («+» и повёрнутый на 45° «×»), `CocktailGlass`, `Bottle` — ЭКСПОРТИРОВАНЫ из `decorations.tsx` и переиспользованы. Hero — по всей ширине; pricing — полоса `min(1340px)` вокруг карточки.
- **Карточка тарифа:** «Авторские коктейли» ↔ «Тесты и шпаргалки» местами; рядом с «Авторские коктейли» — золотая плашка «ЭКСКЛЮЗИВ». Соц-иконки nav 24→30px.
- **АДАПТИВ ДЕКОРА ПОД ULTRAWIDE (важно):** на 2560 фоновый декор жался к центру (края пустые) + разные системы позиционирования уезжали по-разному при ресайзе. Свёл ВСЁ к ЕДИНОЙ вьюпорт-пропорциональной системе: гуттер-иконки = `max(16px, calc((50vw - 550px) * F))` (F=0..1 — доля гуттера; заполняют боковины на любой ширине без наезда на контент 1100px). Hero-глянцевые звёзды — фикс `calc(50% - 512px)` (обрамляют контент). **Правило: НЕ мешать фикс-px / band / % — только пропорционально вьюпорту.**
- **Центровка nav:** блок ссылок был сдвинут ~12–16px влево (`justify-between` + лого 41px vs соцсети 72px). Абсолютно отцентрировал ссылки → «Для кого» встал под фоновой звездой. Сама звезда ВСЕГДА была по центру (проверено на 1000/1280/2560, offset 0) — сдвинута была ШАПКА.

**Состояние:** `tsc --noEmit` чистый; проверено на 1280 и 2560 через Claude-in-Chrome; горизонтального оверфлоу нет.

### Раунд 7 — декор самолётов, центровка шапки, мобильный декор
- **Самолёты about крупнее.** Боковые `.about-plane--left/--right` 150→**185px** (центр остался 130). ВНИМАНИЕ: у `--right` в globals.css СВОЙ `width` (не наследует базу) — правил ДВА места (базу `.about-plane` и `--right`). Левый/правый заякорены за внешний край (`left:-44px` / `right:-40px`), растут ВНУТРЬ под карту → внешние края не двигаются, оверфлоу не появляется; крупнее = больше просвечивает сквозь стекло.
- **Центровка шапки по шпильке звезды.** Блок ссылок был отцентрирован верно (совпадал со звездой), НО средняя ссылка «Для кого» стояла +4.3px правее шпильки: левые ссылки (Главная+О нас) на ~8.5px шире правых (FAQ+Тарифы). Добавил `md:-ml-1` (−4px) блоку ссылок → «Для кого» встала точно под шпильку (offset 0.27px). Коррекция константна на любой ширине (ширины ссылок от вьюпорта не зависят). Звезда/вордмарк/блок-ссылок и так были на центре контент-бокса (offset 0). Остаточный сдвиг всего центрированного контента на ~7.5px влево от ОПТИЧЕСКОГО центра окна — это скроллбар (15px); НЕ трогал (лечится `scrollbar-gutter: stable both-edges`, но это сместит вьюпорт-декор — см. нюанс №6).
- **Мобильный декор переразложен.** Была каша: 17 десктоп-гуттер-иконок на мобиле схлопывались на поля (`max(16px,…)`) в две колонки во всю высоту И дублировались с мобильным набором; hero-иконки на фикс-x (304/336) вылезали на ≤320px. ## Важные технические нюансы (не потерять!)
1. **`preview_screenshot` РАБОТАЕТ (Раунд 7), но эмулятор врёт по ширине.** В прошлых сессиях таймаутил — в Раунде 7 отдавал скрин штатно. НО: `window.innerWidth`/`innerHeight` ≈ **1.8× больше** `document.documentElement.clientWidth`/`Height` (напр. resize 390 → clientWidth 390, а innerWidth 710). Вёрстка/медиазапросы/`50%`/`vw` считаются по clientWidth (390 = мобилка, всё ок), НО СКРИН захватывает более широкую область (710) → в кадр ЛЕЗУТ элементы, которые на реальном телефоне ЗА краем (hero-звёзды `right:calc(50%-512px)` = layout-x≈640 при 390 → на реальном 390 их НЕТ, а на скрине видно яркую звезду справа). **Вывод: для «что видит реальный телефон» верить `getBoundingClientRect` относительно `clientWidth`, НЕ скрину.** Десктоп-ширины без этого артефакта (1280→clientWidth 1265, innerWidth 1280 — обычный скроллбар 15px). Альтернатива для реальных скринов — расширение **Claude-in-Chrome** (у заказчика): `list_connected_browsers`→`select_browser`→`tabs_context_mcp{createIfEmpty}`→`navigate` localhost:3000→`computer{screenshot}` (первый скрин standalone под permission-промпт).
2. **Свежий preview-сервер стартует с 0-шириной вьюпорта** (`innerWidth===0` → мобильная вёрстка, `rowW:0`, огромный `rowH`). После `preview_start` СРАЗУ `preview_resize` явными width/height (1280×800). `preset:"desktop"` НЕ срабатывает — только явные размеры.
3. **`will-change`/`transform` у предка УБИВАЕТ `backdrop-filter` потомков** (Chromium — backdrop берёт пустой фон, блюр = no-op). `.reveal` держал `will-change:transform` → всё стекло было мёртвым. `getComputedStyle().backdropFilter` при этом врёт (показывает blur). Фикс — `will-change:auto` на `.reveal-in`. Диагностика: пройтись по предкам на `transform/filter/perspective/will-change ≠ none/auto`.
4. **Turbopack на этом Windows кеширует устаревший CSS/ошибки.** Правки в `globals.css` могут НЕ доехать до CSSOM (JSX хот-релоадится, CSS стейл). Лечить: `preview_stop` → удалить `.next` (`Remove-Item -Recurse -Force .next`) → `preview_start` (→ снова `preview_resize`!). Проверка: в `preview_eval` искать своё правило в `document.styleSheets`.
5. **Lightning CSS (Tailwind v4) ВЫРЕЗАЕТ `backdrop-filter` из `globals.css`.** Стеклянные блюры задавать `backdropFilter`/`WebkitBackdropFilter` ИНЛАЙН в JSX. Так сделаны `.feature-tag`, about-карточка, тариф.
6. **Декор — ЕДИНАЯ вьюпорт-пропорциональная система.** Гуттер-иконки (`decorations.tsx`): `max(16px, calc((50vw - 550px) * F))`. Hero-декор: проценты; глянцевые звёзды: `calc(50% - 512px)`. НЕ мешать фикс-px/band/% — иначе на ресайзе едут вразнобой + пустые края на ultrawide.
7. **Не запускать `next build` при живом `next dev`** — общий `.next` десинкронизируется.
8. Launch-конфиг превью: `.claude/launch.json`, имя `dev`, порт 3000, `autoPort:true`.
