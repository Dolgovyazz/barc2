## О проекте
Лендинг **BARCODE** — онлайн-библиотека барного дела в Telegram (рецепты коктейлей, виды алкоголя, техники). Язык — русский. Тёмная сине-навигационная «барная» эстетика (чёрный фон, синий `#1a6fff`, голубой `#a8d4ff`, aurora-свечения, глянцевые 3D-звёзды).

- **Путь (корень проекта):** `C:\Users\SystemX\Desktop\barcode\web` — здесь `app/`, `package.json`, отсюда запускать `pnpm dev`. HANDOFF лежит тут же (`web/HANDOFF.md`).
- **Стек:** Next.js 16.2.6 (App Router, **Turbopack**), React 19, Tailwind CSS 4, TypeScript 5.7, pnpm. Аналитика: **Turso** (serverless SQLite) через `@libsql/client`.
- **Запуск:** `pnpm dev` (порт 3000). Проверка: `npx tsc --noEmit`, `npx next build`.
- Много инлайн-стилей (наследие из v0.app) + CSS-переменные в `globals.css`.
- **Деплой:** сайт на **Vercel** из GitHub-репо **`Dolgovyazz/barc2`** (ветка `main`, auto-deploy на пуш). Репо = содержимое папки `web/` (`.git` лежит ВНУТРИ `web/`, поэтому корень репо = `web/`, Root Directory на Vercel по умолчанию).
- **Админ-панель аналитики (Раунды 16–17):** страница **`/admin`** на Vercel со статистикой лендинга (посещения/уникальные/отказы/клики оплаты + графики по времени), БД **Turso**. Это отдельная веб-панель, НЕ Telegram-бот. **ENV на Vercel:** `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN` (ставит интеграция Turso в маркетплейсе), `ADMIN_PASSWORD` (задать вручную!). Локально — `web/.env.local` (в `.gitignore`); схему БД создаёт разово `node --env-file=.env.local scripts/init-db.mjs`.
- **Telegram-бот** (оплаты/доступ через Tribute) — отдельная **Python**-кодовая база в КОРНЕ `C:\Users\SystemX\Desktop\barcode\` (`main.py`, `db.py`, `.env`, `bot.db`…), **НЕ в репо `barc2`**, крутится на своём always-on хосте (aiogram long-polling + aiohttp-вебхук Tribute). Serverless-Vercel для него НЕ подходит. К аналитике на сайте отношения не имеет.

### Раунд 6 — настоящее стекло, декор, адаптив под ultrawide, центровки
- **КРИТИЧНО — блюр стекла НЕ РЕНДЕРИЛСЯ.** `.reveal` (scroll-анимация) держал `will-change: opacity, transform` НАВСЕГДА → в Chromium предок с `will-change`/`transform` делает `backdrop-filter` у потомков no-op (пустой backdrop). Поэтому подъём блюра 14→22→44px не давал НИЧЕГО (заказчик: «как будто не поменялось»). Фикс: `.reveal-ready .reveal.reveal-in { will-change: auto }`. `getComputedStyle` при этом ЛЖЁТ — показывает `blur(44px)`, а на экране ноль. Диагностика: обойти предков на `transform/filter/perspective/will-change ≠ none/auto`.
- **Стекло докручено под референс:** блюры снижены до умеренных (about 12, feature-теги 12, тариф 14, nav 16) + тон стекла сделан полупрозрачным (about `rgba(10,22,46,.62)`+блик), чтобы за стеклом ПРОСВЕЧИВАЛИ фигуры.
- **Самолёты в about переделаны:** убраны декор-звезда + одиночный самолёт; добавлены 3 самолёта (`.about-plane--left/center/right`) слоем за карточками, носами внутрь, крупнее (150/130/150).
- **Свечение на «Бармен»:** радиал из верх-правого угла в `background` карточки + внешний ореол в `box-shadow`.
- **Декор hero и pricing:** `Plus` («+» и повёрнутый на 45° «×»), `CocktailGlass`, `Bottle` — ЭКСПОРТИРОВАНЫ из `decorations.tsx` и переиспользованы. Hero — по всей ширине; pricing — полоса `min(1340px)` вокруг карточки.

- **Причина:** в `package.json` был блок `"pnpm": { "overrides": { "hono": "4.12.25" } }`. Vercel использует **pnpm 10.x** (по дате создания проекта; lockfileVersion `9.0`) — он ЕЩЁ читает `pnpm.overrides` из package.json и ждёт этот override в лок-файле. Но `pnpm-lock.yaml` его НЕ содержит: локальный **pnpm 11.9.0** (corepack) этот блок ИГНОРИРУЕТ (в pnpm 10+ overrides переехали в `pnpm-workspace.yaml`) и при последней регенерации записал лок без overrides. Config(pnpm10)={hono} ≠ lockfile(none) → frozen install падает.
- **Фикс:** удалил мёртвый блок `pnpm` из `package.json` (коммит `33bce75`). Теперь overrides нет нигде → совпадает с лок-файлом. `hono` остаётся `4.12.25` (транзитивно залочен через `@hono/node-server`, резолв не меняется). Проверено: `pnpm install --frozen-lockfile` → «Already up to date», exit 0, лок-файл не тронут.
- **NB на будущее:** если override реально понадобится — класть в `pnpm-workspace.yaml` (секция `overrides:`), НЕ в `package.json` (pnpm 11 его там не видит). Фикс лечит шаг установки; `next build` на Vercel отдельно (локально не гоняли — нельзя при живом dev, нюанс №7).

**Состояние:** `tsc --noEmit` чистый; Vercel-install чинён (мёртвый `pnpm.overrides` убран, frozen install проходит); всё запушено в `barc2`.

## Важные технические нюансы (не потерять!)

> **Превью-тулы менялись между сессиями.** Были `mcp__Claude_Preview__preview_*` (`preview_eval/resize/screenshot/stop/start`); в Раунде 11 стали `mcp__Claude_Browser__*` (`javascript_tool` = eval, `resize_window`, `computer{action:screenshot}`, `navigate`, `preview_start{name:"dev"}` / `preview_stop{serverId}` / `preview_logs`). Логика та же, имена разные — сверяйся со списком тулов в системном промпте.

1. **Скриншоты НЕНАДЁЖНЫ — мерить через JS (`javascript_tool`/`preview_eval`), а не глазами по скрину.** Два бага: (а) `computer{screenshot}` в Раунде 11 РАБОТАЛ только вверху страницы; ПОСЛЕ прокрутки (`scroll`/`scroll_to`) стабильно ТАЙМАУТИЛ 30с или отдавал ЧЁРНЫЙ кадр (пейн залипает; лечится `navigate` заново, но скролл сбрасывается) → секции ниже hero (about/for-whom) заскринить не удавалось. (б) Эмулятор врёт по ширине: `window.innerWidth` ≠ `clientWidth` (наблюдалось 1.8×, напр. 710 vs 390) → скрин захватывает БОЛЬШЕ, чем реальный телефон, и в кадр лезут элементы, которые на реальном 390 ЗА краем (hero-звёзды `calc(50%-512px)`). **Вывод: судить о «реальном телефоне» по `getBoundingClientRect` относительно `clientWidth`, НЕ по скрину.** Десктоп-ширины артефакта не имеют (1280→clientWidth 1265). Реальные скрины — расширение **Claude-in-Chrome** (у заказчика).
2. **Свежий preview-сервер стартует с 0-шириной вьюпорта** (`innerWidth===0` → мобильная вёрстка, `rowW:0`, огромный `rowH`). После `preview_start` СРАЗУ `preview_resize` явными width/height (1280×800). `preset:"desktop"` НЕ срабатывает — только явные размеры.
3. **`will-change`/`transform` у предка УБИВАЕТ `backdrop-filter` потомков** (Chromium — backdrop берёт пустой фон, блюр = no-op). `.reveal` держал `will-change:transform` → всё стекло было мёртвым. `getComputedStyle().backdropFilter` при этом врёт (показывает blur). Фикс — `will-change:auto` на `.reveal-in`. Диагностика: пройтись по предкам на `transform/filter/perspective/will-change ≠ none/auto`.
4. **Turbopack на этом Windows кеширует устаревший CSS.** Правки в `globals.css` могут НЕ доехать до CSSOM (JSX хот-релоадится, CSS стейл). Лечить: остановить dev-сервер (`preview_stop`) → удалить `.next` (`rm -rf .next` в Bash-туле, ИЛИ `Remove-Item -Recurse -Force .next`) → снова запустить (`preview_start{name:"dev"}`) → СРАЗУ `resize_window`/`preview_resize` (иначе стартует с 0/мобильной шириной — нюанс №2). Каждую правку `globals.css` в этой сессии гоняли через этот цикл. Проверка: `getComputedStyle` элемента показывает новое значение. JSX/инлайн-стили (`page.tsx`, `nav.tsx`, `pricing.tsx`) хот-релоадятся без вайпа.
5. **Lightning CSS (Tailwind v4) ВЫРЕЗАЕТ `backdrop-filter` из `globals.css`.** Стеклянные блюры задавать `backdropFilter`/`WebkitBackdropFilter` ИНЛАЙН в JSX. Так сделаны `.feature-tag`, about-карточка, тариф.
6. **Декор — ЕДИНАЯ вьюпорт-пропорциональная система.** Гуттер-иконки (`decorations.tsx`): `max(16px, calc((50vw - 550px) * F))`. Hero-декор: проценты; глянцевые звёзды: `calc(50% - 512px)`. НЕ мешать фикс-px/band/% — иначе на ресайзе едут вразнобой + пустые края на ultrawide.
7. **Не запускать `next build` при живом `next dev`** — общий `.next` десинкронизируется.
8. Launch-конфиг превью: `.claude/launch.json`, имя `dev`, порт 3000, `autoPort:true`.
